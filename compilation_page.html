To set up the case one has to check module dependency of the main tool described in FRAMEWORK.txt. An example shows FRAMEWORK.txt file from time stepper module:

#List of existing submodules
SUBMODULES arnildi_arpack \
powerit
#List of conflicting submodules
EXCLUSIVE arnoldi_arpack \
powerit
#Default submodules
DEFAULT arnoldi_arpack
#List of conflicting modules
CONFLICTS
#List of required modules
REQUIRED
#List of object files
OBJECT time_stepper.o
#List of include files
INCLUDE TIME_STEPPERD

This module contains two conflicting submodules Arnoldi algorithm using parpack and Power itertion routines. To setup specific case one has to check recursively dependency of all specified modules to get a complete list of required modules and exclude possible conflicts. In the case of spectra calulation with PARPACK library one checks FRAMEWORK.txt under tools/tstepper/arnoldi_arpack/ and finds that arnoldi_arpack depends on io/checkpoint/dummy and io/io_tools. On the other had io/checkpoint/dummy depends on io/checkpoint, so the final list of equired modules is:

  • tools/tstepper,
  • tools/tstepper/arnoldi_arpack/,
  • io/checkpoint,
  • io/checkpoint/dummy,
  • io/io_tools.

Next one copies to the setup directory the source files listed in OBJECT and INCLUDE sections of FRAMEWORK.txt of all required modules, and adds content of OBJECT to USR variable in makenek. The last step before compilation is to concatenate makefile_usr.inc from requred modules' directories to generate makefile_usr.inc in the setup directory.
